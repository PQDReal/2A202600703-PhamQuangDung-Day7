Recursive Chunker

Recommended

Ưu tiên separator theo thứ tự: paragraph → dòng → câu → khoảng trắng. Cân bằng nhất.

RecursiveChunker(
  separators=["\\n\\n","\\n",". "," ",""],
  chunk_size=500
)
✅ Giữ cấu trúc paragraph/heading
✅ Fallback thông minh
✅ Retrieval quality cao nhất

📐 Công Thức Tính Số Chunk

chunks = ⌈(doc_length − overlap) / (chunk_size − overlap)⌉
Ví dụ 1: doc=10,000 chars, size=500, overlap=50
⌈(10000 − 50) / (500 − 50)⌉ = ⌈9950/450⌉ = ⌈22.1⌉ = 23 chunks
Ví dụ 2: doc=10,000 chars, size=500, overlap=100
⌈(10000 − 100) / (500 − 100)⌉ = ⌈9900/400⌉ = ⌈24.75⌉ = 25 chunks

Key Insight #1

Mock embeddings không hiểu ngữ nghĩa thật. Pipeline hoạt động nhưng score không phản ánh similarity. Cần all-MiniLM-L6-v2 hoặc OpenAI cho semantic evaluation nghiêm túc.

🔍

Key Insight #2

Metadata filtering quan trọng không kém vector similarity. search_with_filter(language=en) loại được chunks tiếng Việt trước khi rank, cải thiện precision đáng kể.

🧩

Key Insight #3

RecursiveChunker là best default cho tài liệu .md/.txt có cấu trúc. Nó tận dụng paragraph trước, fallback câu/từ sau — preserving context tốt nhất trong benchmark.

🤖

Key Insight #4

RAG agent cần grounding prompt rõ ràng: "chỉ dùng context dưới đây, nói rõ nếu không đủ thông tin". Điều này giảm hallucination và trace kết quả về nguồn gốc.
